# Анализ качества FASTQ и фильтрация ридов для RNA-seq

## Описание проекта

При секвенировании ДНК или РНК (RNA-seq) прибор выдаёт данные в формате FASTQ. Каждый рид содержит:

- Последовательность нуклеотидов (A, T, G, C)
- Строку качеств (Phred score) — числовое значение, показывающее, насколько прибор уверен в правильности каждого нуклеотида

**Проблема:** В сырых данных всегда есть "шум" — риды с низким качеством, короткие фрагменты, технические артефакты.

**Цель проекта:**
- Научиться парсить FASTQ-файл
- Рассчитать статистику качества для каждого положения в риде
- Отфильтровать риды по длине и качеству
- Сохранить "чистые" данные для дальнейшего анализа
- Визуализировать результаты фильтрации

## Параметры фильтрации

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| `min_length` | 50 нт | Минимальная длина рида |
| `min_quality` | 25 | Порог Phred score (99.7% точность) |
| `min_percent_good` | 85% | Требуемая доля хороших позиций |

### Визуализации

Программа создаёт два графика в папке `results/`:  

1. **`quality_plot.png`** — распределение качества по позициям в риде  
   - Синяя область: 25-75 перцентиль  
   - Синяя линия: медиана (Q50)  
   - Красный пунктир: среднее значение  
   - Цветные линии: пороги качества (Q20, Q25, Q30)  

2. **`filtering_stats.png`** — статистика фильтрации  
   - Количество ридов до и после фильтрации  
   - Причины отсева (длина / качество)  

## Выводы

1. **Качество данных хорошее** — средний Phred score >30 по всем позициям  
2. **Проблемные позиции** обнаружены в конце ридов (97-100) — рекомендуется тримминг  
3. **Сохранено 75% ридов** — достаточно для дальнейшего анализа  
4. **Очищенные данные** (`data/filtered_reads.fastq`) готовы к выравниванию на референсный геном
